1 Host bridge daemon ~2 hrs ▼

Lightweight Python daemon on macOS — never in Docker. Owns all hardware + OS access. Streams audio and screen to container; executes JSON action payloads back.

  • 1
    Audio capture
    pvrecorder streams mic frames (16 kHz, 16-bit mono) over WebSocket to the container's /transcribe endpoint. Docker cannot touch the mic — this lives only on the host.
  • 2
    Screen context
    On each command cycle: screenshot via Quartz.CGWindowListCreateImage, resize to 1024×768 (sufficient for UI detection, saves vision tokens), send as base64 alongside transcript. Active app name + window title attached as metadata.
  • 3
    Action executor — two strategies
    def execute(action):
    v = action["verb"]
    if v == "applescript":
        subprocess.run(["osascript", "-e", action["script"]])
    elif v == "click_ax":              # Strategy 1: AX tree (fast, no vision)
        ax_click_by_label(action["label"], action.get("role","AXButton"))
    elif v == "click_xy":              # Strategy 2: vision fallback (Electron/web)
        pyautogui.moveTo(action["x"], action["y"], duration=0.08)
        pyautogui.click()
    elif v == "type":
        pyautogui.typewrite(action["text"], interval=0.03)
    elif v == "hammerspoon":
        subprocess.run(["hs", "-c", action["lua"]])
    elif v == "shell":
        subprocess.run(action["cmd"], shell=True)

2 Container brain — STT + intent agent ~3 hrs ▼

Docker handles all cognition. Calls Ollama at host.docker.internal:11434 — Metal GPU stays on the host where macOS allows it. All tuning values injected as env vars.

  • 1
    Dockerfile
    FROM python:3.12-slim
RUN pip install --no-cache-dir \
    faster-whisper fastapi uvicorn websockets \
    ollama httpx pillow
COPY . /app
WORKDIR /app

  • 2
    STT — faster-whisper
    Receives raw audio bytes, transcribes with device-tuned model size. Returns transcript + word-level confidence. Confidence directly feeds OODA Gate 1.

model = WhisperModel(
    os.environ["WHISPER_MODEL"],   # base
    device="cpu",
    compute_type="int8",
    num_workers=4
)
  • 3
    Gemma 4 intent + vision call
    Single call: transcript + screenshot → structured action JSON via native function calling. No regex parsing. Vision token budget controls latency vs accuracy.

3 OODA loop + ghost mode ~4 hrs ▼

Decision layer. Ghost mode = silent by default. Surfaces only when all three gates pass — no interruption without value.

  • 1
    Observe
    Receives rolling audio + screen from bridge. Maintains sliding window of last CTX / 4 tokens of history — auto-sized to device context window.
  • 2
    Orient
    Gemma 4 call with full context. Returns intent + confidence 0–1. Below 0.85 → stay in loop. Active app injected as system prefix so context-dependent commands resolve correctly.
  • 3
    Decide — 3 serial gates
    SILENT_TASKS = {"file_index","file_read","context_update"}

def should_surface(intent) -> bool:
    if intent["confidence"] < 0.85:           # Gate 1: confidence
        return False
    if not intent.get("urgent") and \
       intent["task_type"] in SILENT_TASKS:   # Gate 2: urgency
        return False
    if not verifier.check(intent):            # Gate 3: completeness
        queue_for_human_review(intent)
        return False
    return True
  • 4
    Act
    Sends JSON action to host bridge. Writes result to lossless event log (append-only JSONL with model name + ctx_used). Publishes to /artifacts only when gates pass.
  • 5
    Implicit fast-path
    Patterns executed successfully ≥3 times skip all gates: Orient → Act directly. Stored in state/learned_patterns.json. First execution always runs all three gates.

4 Wake word + manual verifier ~2 hrs ▼
  • 1
    Wake word — two options
    Option A (start here): Hammerspoon global hotkey (⌥Space) — zero latency, zero CPU, zero ML. Option B: Porcupine free tier — one custom keyword, ~1% CPU, 20–40 ms detection.
    pip install pvporcupine pvrecorder

-- Hammerspoon init.lua (Option A)
hs.hotkey.bind({"alt"}, "space", function()
  hs.urlevent.openURL("ghost://listen")
end)
  • 2
    Manual verifier UI
    Triggers when STT confidence is 0.70–0.84. Fires a macOS notification with action buttons: "Did you say: open Spotify?" — Yes / No / Retry. Yes bypasses retranscription. Retry re-runs the audio segment through Whisper base.
